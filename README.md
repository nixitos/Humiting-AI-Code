# Humiting AI Code

Инструмент для приведения кода, сгенерированного LLM, к читаемому и естественному виду, как если бы его написал человек-программист.

## О проекте

Humiting AI Code — это набор промптов для LLM (таких как ChatGPT, DeepSeek, Grok), которые изменяют стиль ответа модели. Вместо "академичного" и многословного кода вы получаете лаконичный, структурированный и "человечный" код, написанный в духе известных программистов.

Инструмент не требует установки, работает в любом новом чате с LLM и не меняет логику вашего кода — только его внешний вид и стиль.

## Как это работает

Вы отправляете LLM промпт из репозитория, а следом — свой код. LLM читает промпт по ссылке и применяет его правила к вашему коду. Результат вы получаете в том же чате.

## Стили

В репозитории доступны несколько предустановленных стилей. Выберите тот, который ближе вам по духу:

| Стиль | Файл | Описание |
|-------|------|----------|
| Базовый (универсальный) | `main-prompt.txt` | Стандартный стиль, сокращает имена переменных и функций, убирает лишние комментарии |
| Carmack | `carmack-style.txt` | Максимально короткие имена (`x`, `y`, `p`), агрессивные сокращения функций (`calc()`, `upd()`), почти без комментариев. Вдохновлён стилем Джона Кармака |
| Flowseal | `flowseal-style.txt` | Внутренние функции с префиксом `_`, короткие, но осмысленные имена (`cfg`, `exc`). Комментарии только по делу |
| Torvalds | `lunus-torvalds-style.txt` | Имена переменных 3-5 символов (`buf`, `cnt`, `pos`), функции с подчёркиваниями (`get_data()`). Стиль комментариев как в ядре Linux |

Вы можете использовать любой из этих файлов как промпт. Если хотите создать свой стиль — просто скопируйте любой файл и измените правила под себя.

## Как использовать

1. Откройте новый чат с LLM, которая поддерживает чтение контента по ссылкам (ChatGPT, DeepSeek, Grok, и другие).
2. Отправьте первым сообщением ссылку на файл с нужным стилем. Например, для базового стиля:
   ```
   https://raw.githubusercontent.com/nixitos/Humiting-AI-Code/refs/heads/main/main-prompt.txt
   ```
3. Следующим сообщением отправьте ваш код.
4. LLM применит правила из промпта и вернёт обработанный код.

Дополнение: Применив промпт можно вайбкодить и LLM будет сохранять стиль кода

## Пример

**Исходный код (сгенерирован LLM):**
```python
def update_player_score(player_object, score_amount):
    # This function updates the player's current score by adding the provided amount
    player_object.score += score_amount
    return player_object.score
```

**После обработки базовым стилем:**
```python
def updPlr(plr, amt):
    # add score
    plr.score += amt
    return plr.score
```

**После обработки стилем Carmack:**
```python
def upd(p, a):
    p.sc += a
    return p.sc
```
